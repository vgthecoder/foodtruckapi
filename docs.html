Food Truck Api: Docs

Endpoints

The Food Truck Api has one endpoint, at https://foodtruckapi.onrender.com/.

To request items, you will need to use ?type and &query.

This Api is NOT case-sensitive. (to be fixed) The API may take up to 50 seconds to spit something out.

TypeResult
(A)ll DishesWill return all the dishes that are in the database.
(L)evelWill return all the dishes that are in the level listed after the &query=.
(I)ingredientWill return all the dishes that contain that ingredient listed after &query=.
(E)xact NameWill return one dish who's exact name is listed after &query=.
(N)ameWill return all the dishes who's name, or part of it, is listed after &query=.

Example Request Url, to get all dishes that have dough in them:

https://foodtruckapi.onrender.com?type=i&query=dough

Example Request Url, for Figure 1

https://foodtruckapi.onrender.com/?type=e&query=croissant

Responses

Expected responses from the Food Truck Api are in json. Their structure is listed below. (Figure 1)

KeyDescription
errWill return "" if there are no errors. Will describe the error if there is one.
resultsWill return the number of dishes found. Will return 0 if none were found.
dishesWill return an array with the keys being the names of the dishes that were found.
> ico0Will return a unicode emoji corresponding to that dish. Will show up as a ‚ùì if an emoji for it does not exsist in unicode.
> catWill return the catagory of the dish. It will be one of the following: Basic I II III Exotic Secret
> ingXWill return the name of the Xth ingredient in the dish.
> icoXWill return a unicode emoji of the Xth ingredient in the dish. Will show up as a ‚ùì if an emoji for it does not exsist in unicode.
